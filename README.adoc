= Base App

== Goal

The goal of this project is to provide a basic app running in a web browser to manipulate data of a database persisting
a business domain model. +
My goal was to use the simplest possible frameworks/architectures, each leveraging the native technologies of the
components used. This should result in very easily maintainable code.

=== Development environment

As development environment I use Docker to locally run the needed services - namely PostgreSQL and Keycloak.

=== Backend

To ensure the backend can be scaled as painlessly as possible, it should remain stateless.

==== Spring Boot with Kotlin

- Spring Boot with Kotlin
- Gradle build system with Kotlin DSL
- Spring Boot Data JDBC
- Flyway Migration
- Spring-Boot-Security with Keycloak
- Spring-Boot-Actuator
- Spring-Boot-DevTools
- SpringDoc OpenAPI
- Apache Freemarker
- Testcontainers for testing with PostgreSQL and Keycloak

I wanted to avoid using JPA. In my opinion, this architecture tends to implement entities that tend to load too many
related objects in SQL queries. +
By using Kotlin, it is very easy to implement lean entity classes for each query, which can be easily loaded and
manipulated via the Spring Data repository architecture.

==== Keycloak

Keycloak is used for user and permissions management using JWT tokens in the backend.

=== Frontend

Here too, the goal is to use the leanest possible architecture. I wanted to avoid using a frontend framework such as
Angular, React, or Vue. So no more complexity by using a frontend framework if it is not really needed.

- Bootstrap for the frontend
- Bootstrap Table for rendering dynamic tables with data fetched via a ReST API from the backend
- Bootstrap Icons for icons in the frontend

The project usees server-side rendering with Apache Freemarker templates to encapsulate the boilerplate HTML code
to create HTML components with Bootstrap.

This project contains a docker-compose.yaml (`local/docker/...`) file that defines the following services:

- PostgreSQL (port: 5432)
- http://localhost:8082/admin/master/console/#/baseapp[Keycloak - realm BaseApp] (port: 8082)

==== PostgreSQL

The PostgreSQL database persists its data in the folder postgresql directly besides the docker-compose.yaml file.
So you are able to delete the complete database by simply deleting the content of the postgresql folder.

PostgreSQL server is configured to have different databases for the BaseApp and Keycloak.

===== Initial setup of the databases `baseapp` and `keycloak`:

In `docker-compose.yaml` the image `postgres:18` is used for the PostgreSQL server with its well-known environment
parameters to configure the initial database. +
Using these parameters the `keycloak` database is created with owner/role `keycloak` and the password `keycloak`. +
The image allows you to execute SQL scripts at startup that are found in the containers folder
`/docker-entrypoint-initdb.d`. This folder is linked to `local/docker/docker-entrypoint-initdb.d` in the project,
containing the script `postgresql-init.sql`. +
As PostgreSQL does not have the command `CREATE DATABASE IF NOT EXISTS ...`, I had to use some _tricks_ creating the
second database for the BaseApp. +
The extension `dblink` adds the feature to connect to a PostgreSQL and execute SQL statements there. So the script
connects to the already created `keycloak` database and simply executes `CREATE DATABASE baseapp` there.
If the database already exists the `duplicate_database` exeption is handled to continue the execution of the script. +
Then the role `baseapp` is created if not exists set as owner to the `baseapp` database.

===== PostgreSQL databases and users:

* database `keycloak` with owner/role `keycloak` and password `keycloak`
* database `baseapp` with owner/role `baseapp` and password `baseapp`
